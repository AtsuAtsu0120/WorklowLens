# tool_logger プロジェクト

## 仕様書駆動設計 (SDD)

このプロジェクトは**仕様書駆動設計（Spec-Driven Design）**を前提とする。

### 基本ルール

1. **実装前に仕様書を確認する** — 機能の実装・修正を依頼されたら、まず `Document/` 配下の該当する仕様書を読む
2. **仕様書がなければ先に作成する** — 該当する仕様書が存在しない場合、実装に入る前に仕様書を作成し、ユーザーの承認を得る
3. **実装後に仕様書を更新する** — 実装の結果、仕様と差異が生じた場合は仕様書を更新する
4. **仕様書が正（Single Source of Truth）** — コードと仕様書に矛盾がある場合、仕様書に合わせるのが原則。仕様書側を変えるべき場合はユーザーに確認する

### ディレクトリ構成

```
各サブプロジェクト/
  Document/
    overview.md          全体概要・アーキテクチャ
    features/            機能別の仕様書
```

### 仕様書フォーマット

- Markdown形式
- 日本語で記述
- 機能単位で1ファイル（`Document/features/` 配下）

## プロジェクト構成

- `tool_logger_middleware/` — UDP受信→HTTP転送の中継サーバー（Go）
- `tool_logger_server/` — ログ保存・閲覧のWebサーバー（Go）
- `tool_logger_client_csharp/` — UDP送信クライアントライブラリ（C# / .NET Standard 2.1）
- `tool_logger_client_python/` — UDP送信クライアントライブラリ（Python 3.7+）

## 共通規約

- コメント・ドキュメントは日本語
- ユーザーはC#に精通、Goもある程度わかる。冗長な言語基礎の説明は不要
