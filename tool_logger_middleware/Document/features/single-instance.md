# 多重起動防止

## 概要

tool_loggerの多重起動を防止し、同一マシンで常に1インスタンスのみ動作することを保証する。

## 背景・目的

tool_loggerはローカルの中継ツールであり、同じマシンで複数起動する必要がない。誤って2つ目を起動すると、同じポートの取り合いやログの分散が発生する。現状は `TcpListener::bind` の `AddrInUse` エラーが出るだけで、ユーザーにとって原因が分かりにくい。また、メインポートを変えて起動した場合は検出できない。

## 要件

- [ ] 同じマシンで2つ目のtool_loggerを起動しようとしたとき、分かりやすいエラーメッセージを表示して終了する
- [ ] メインポートを変えても多重起動を検出できる（例: 1つ目が59100、2つ目が59200）
- [ ] プロセスが異常終了（クラッシュ、kill）した場合、ロックが自動解放される（手動クリーンアップ不要）
- [ ] `std`のみで実装する（追加クレートなし）

## 設計

### 方式: ロックポート方式

メインポートとは別に、ロック専用のTCPポート（デフォルト: 59099）をバインドする。2つ目のインスタンスが同じポートをバインドしようとすると `AddrInUse` で失敗するため、「既に起動中」と判定できる。

**C#との対比**: C#での多重起動防止は `Mutex` を使うのが一般的:

```csharp
using var mutex = new Mutex(true, "tool_logger", out bool createdNew);
if (!createdNew)
{
    Console.Error.WriteLine("既に起動中です");
    return;
}
```

Rustの標準ライブラリには名前付きMutexがないため、「ポートのバインド」を排他制御の手段として代用する。ポートはプロセス終了時にOSが自動解放するため、PIDファイルのような「クラッシュ後にゴミが残る」問題がない。

### 他方式との比較

| 方式 | クラッシュ耐性 | 追加crate | コード量 |
|------|-------------|-----------|---------|
| **ロックポート（採用）** | OSが自動解放 | 不要 | 少ない |
| PIDファイル+ファイルロック | `libc`のflock必要 | `libc` or `fs2` | 多い |
| PIDファイル（ロックなし） | ゴミファイルが残る | 不要 | 中程度 |

### モジュール構成

```
src/instance_lock.rs  → InstanceLock構造体、ロック取得/解放
src/main.rs           → サーバー起動前にロック取得
src/error.rs          → AlreadyRunningバリアント追加
```

### InstanceLock (`src/instance_lock.rs`)

```rust
/// インスタンスロック（C#の named Mutex に相当）
pub struct InstanceLock {
    _listener: TcpListener,  // Dropされるとポートが解放される
}
```

- `acquire()` — デフォルトポート（59099）でロック取得
- `acquire_on_port(port: u16)` — 指定ポートでロック取得（テスト用）

**Rustの学習ポイント: `_lock` vs `_`**

```rust
let _lock = InstanceLock::acquire()?;  // main()終了まで保持される
let _ = InstanceLock::acquire()?;      // 即座にDropされ、ロックが解放される！
```

`let _lock` は「使わないけど保持する」、`let _` は「即座に捨てる」。これはRustの所有権・Dropの重要なポイント。C#でいうと前者は `using var mutex = ...`、後者は `new Mutex(...)` の戻り値を変数に入れないのに近い。

### エラー型

`AlreadyRunningError` — 既に別インスタンスが起動中であることを示す専用エラー。

エラーメッセージ:
```
tool_loggerは既に起動中です。複数のインスタンスを同時に実行することはできません。
```

### エントリポイント (`src/main.rs`) の変更

```rust
fn main() {
    // 1. 多重起動防止ロックを取得
    // 2. コマンドライン引数からポートを取得
    // 3. サーバー起動
}
```

ロック取得はサーバー起動より前に行う。

## テスト方針

- [ ] ユニットテスト: ロックを取得できる
- [ ] ユニットテスト: 二重ロックはエラーになる
- [ ] ユニットテスト: ロック解放後に再取得できる
- [ ] 手動テスト: ターミナル2つでの動作確認

テストではポート0（OS自動割り当て）を使用し、テストの並列実行で競合しないようにする。
